# x264video4osu

[![en](https://img.shields.io/badge/lang-en-blue.svg)](README.md) [![zh](https://img.shields.io/badge/lang-zh-red.svg)](README.zh-CN.md)

一个用于编码 osu! 背景视频的工具。

![License](https://img.shields.io/badge/license-MIT-blue.svg)
![Platform](https://img.shields.io/badge/platform-Windows-lightgrey.svg)
![Framework](https://img.shields.io/badge/.NET-10.0-purple.svg)

## 动机

小丸工具箱停止更新，不再支持 H.265 或 AV1。我需要一键压制 osu 视频的工具，于是写了这个。

这只是一个 FFmpeg 前端，用来解决我个人的压视频问题。实际上用命令行就可以实现。需要更多压视频功能的话有很多开源的 ffmpeg 前端可以用。

**注意**：本项目为全 vibe coding 完成，不保证质量，仅供娱乐使用。

## 功能

- 支持 **2-Pass VBR** 和 **CRF** 两种编码模式
- 实时编码进度显示（进度百分比、耗时、预计剩余时间）
- 自定义输出分辨率（支持缩放）
- 可选音轨提取
- 支持中文和英文界面
- 详细的编码日志记录和导出
- 支持拖拽文件输入

## 下载与安装

### 前置要求

需要以下文件放置在 `tools` 文件夹下：

- `ffmpeg.exe`
- `ffprobe.exe`

可以从 [FFmpeg 官网](https://ffmpeg.org/download.html) 下载，或使用 [gyan.dev](https://www.gyan.dev/ffmpeg/builds/) 的构建版本。

### 使用步骤

1. 将 `ffmpeg.exe` 和 `ffprobe.exe` 放入 `tools` 文件夹
2. 运行 `x264video4osu.exe`
3. 选择输入视频文件
4. 配置编码参数（或使用默认值）
5. 点击 "开始" 进行编码

## 编码模式说明

程序提供两种主要的编码模式，各有优劣：

### 2-Pass VBR 模式（推荐用于 osu! 背景）

**工作原理**：第一遍扫描视频分析复杂度，第二遍根据分析结果分配比特率。

| 项目 | 说明 |
|------|------|
| **控制方式** | 固定目标比特率（单位：kbps） |
| **默认值** | 800 kbps |
| **输出大小** | 可精确控制（比特率 × 时长 ≈ 文件大小） |
| **编码速度** | 较慢（需要两遍扫描） |
| **适用场景** | 对文件大小有严格要求的项目 |

**优势**：
- ✅ 文件大小可预测且精确控制
- ✅ 复杂场景分配更多比特，简单场景节省比特

**劣势**：
- ❌ 编码时间约是单次编码的 2 倍
- ❌ 配置相对复杂（需要估算合适的比特率）

**文件大小估算示例**（以 800 kbps 为例）：

| 时长 | 预估文件大小 |
|------|--------------|
| 1 分钟 | ~6 MB |
| 1.5 分钟 | ~9 MB |
| 2 分钟 | ~12 MB |
| 2.5 分钟 | ~15 MB |
| 3 分钟 | ~18 MB |

---

### CRF 模式（Constant Rate Factor）

**工作原理**：恒定质量因子编码，质量优先于文件大小。

| 项目 | 说明 |
|------|------|
| **控制方式** | CRF 值（范围 0-51） |
| **默认值** | 26 |
| **数值意义** | 值越小质量越好，文件越大；值越大质量越差，文件越小 |
| **输出大小** | 不可预测（取决于视频内容复杂度） |
| **编码速度** | 快（单次编码） |
| **适用场景** | 质量优先，文件大小要求宽松的项目 |

**CRF 值参考**：

| CRF 值 | 质量 | 说明 |
|--------|------|------|
| 18-23 | 较高 | 接近无损，文件较大 |
| 23-26 | 中高 | 适合一般用途 |
| 26-28 | 中等 | 适合一般用途，文件可控 |
| 28-32 | 较低 | 适合低带宽场景 |
| 32+ | 低 | 不推荐用于背景视频 |

**优势**：
- ✅ 编码速度快
- ✅ 配置简单（只需设定一个质量值）

**劣势**：
- ❌ 无法精确控制文件大小
- ❌ 对于复杂场景可能超出 osu! 文件大小限制

---

## 推荐配置（osu! 背景视频）

对于典型的 osu! 背景视频（2 分钟以下，要求 < 15MB），推荐使用以下配置：

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 编码模式 | **2-Pass** | 精确控制文件大小 |
| 比特率 | **800-1000 kbps** | 根据视频长度调整 |
| 分辨率 | **720p 或原分辨率** | 保持竖屏比例 |
| FPS | **24** | 帧率设置 |

**比特率选择建议**：

- 视频时长 1 分钟以内：`800-1000 kbps`
- 视频时长 1-1.5 分钟：`700-900 kbps`
- 视频时长 1.5-2 分钟：`600-800 kbps`

## 使用方法

### 基本操作流程

1. **选择输入视频**
   - 点击 "浏览" 按钮选择文件
   - 或直接将视频文件拖拽到窗口上

2. **设置输出路径**（可选）
   - 不设置则自动在输入文件同目录生成 `{原文件名}_output.mp4`

3. **配置编码参数**

   | 参数 | 说明 |
   |------|------|
   | **编码标准** | 选择 2pass 或 CRF 模式 |
   | **数值框** | 2pass 为比特率，CRF 为质量值 |
   | **分辨率** | 宽×高，设为 0 表示保持原值 |
   | **Scale Up** | 允许放大（默认关闭） |
   | **Extract Audio** | 单独提取音轨为 .m4a 文件 |

4. **点击 "开始"** 启动编码

5. **查看进度**
   - 主界面显示实时进度
   - "Log" 标签页可查看详细日志

6. **完成处理**
   - 显示输出文件大小
   - 可点击 "Open Output Folder" 打开输出目录

### 日志功能

- 编码过程实时显示在 Log 标签页
- 可通过 "Save Log" 按钮保存完整日志
- 日志文件命名格式：`log_yyyyMMddHHmmss_随机数.txt`

## 技术参数说明

### 默认编码参数

程序使用以下固定参数：

| 参数 | 值 | 说明 |
|------|-----|------|
| 编码器 | libx264 | H.264 编码器 |
| 预设 | veryslow | 编码较慢 |
| 配置文件 | high | 高配置文件 |
| 级别 | 5.2 | 高级别配置 |
| 像素格式 | yuv420p | 广泛兼容的像素格式 |
| GOP 大小 | 300 | 最大关键帧间隔 |
| 最小 GOP | 240 | 最小关键帧间隔 |
| B 帧 | 16 | 高 B 帧数量 |
| 运动估计 | UMH | UMH 运动估计 |
| 子像素细分 | 11 | Level 11 |

### 高级 x264 参数

```
scenecut=0, ref=16, bframes=16, b-adapt=2, direct=auto,
me=umh, subme=11, trellis=2, rc-lookahead=60, aq-mode=3,
aq-strength=1.0, psy-rd=1.0,0.15, deblock=-1,-1,
weightp=2, cabac=1, merange=32
```
